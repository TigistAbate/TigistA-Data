# World Database ERD

## Overview

This project was completed during my Level 3 Data Technician Skills Bootcamp using MySQL Workbench. I created an Entity Relationship Diagram (ERD) to understand how tables in a relational database are connected through primary and foreign keys.

## Objective

My objective was to design and explore a relational database by creating an Entity Relationship Diagram (ERD) that clearly shows the relationships between tables in the World database.

## Skills Demonstrated

- MySQL Workbench
- Database design
- Entity Relationship Diagrams (ERDs)
- Relational database modelling
- Primary keys
- Foreign keys
- One-to-many relationships
- Data modelling

## Dataset

I used the **World** database, which contains information about countries, cities and languages. I used this dataset to design an Entity Relationship Diagram (ERD) that demonstrates how tables are related within a relational database.

## Organisation Type

This type of database design could be used by government organisations, international organisations, research institutions or businesses that manage large amounts of structured data.

## Business Value

A well-designed relational database helps organisations organise data efficiently, reduce duplication and maintain data integrity. Understanding table relationships also makes it easier to retrieve, manage and analyse data.

## What I Did

- I explored the World database in MySQL Workbench.
- I identified the tables and examined how they were related.
- I created an Entity Relationship Diagram (ERD) to represent the database structure.
- I identified the primary keys and foreign keys used to connect the tables.
- I modelled one-to-many relationships between the tables.
- I reviewed the ERD to ensure the relationships were accurate and clearly represented.

## Key Findings

- Creating the ERD helped me understand how relational databases organise and connect data.
- Identifying primary and foreign keys showed how different tables are linked together.
- One-to-many relationships allow related information to be stored efficiently while reducing data duplication.
- Designing the ERD before writing SQL queries gave me a clearer understanding of the database structure and how data can be retrieved.
